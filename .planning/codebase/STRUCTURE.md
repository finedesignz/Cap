## Directory Purposes

**`apps/web` (deploy target):**
- Purpose: Next.js App Router app — public site, dashboard, share pages, API, auth.
- Key files: `next.config.mjs`, `Dockerfile`, `instrumentation.ts`, `app/`, `actions/`, `lib/`, `vercel.json`.

**`apps/web/app`:**
- Route groups: `(site)` marketing/SEO, `(org)` dashboard+auth, `(docs)`, `admin`, `s/` shares, `embed`, `c/` collections.
- `api/`: `[[...route]]`, `erpc`, `auth`, `upload`, `storage`, `video(s)`, `webhooks`, `cron`, `desktop`, `developer`.

**`apps/web/actions`:** server actions (billing, videos, organization, folders, spaces…).

**`packages/database`:** `schema.ts`, `index.ts` (`db()` MySQL client), `migrations/`, `migrate.ts`.

## Key File Locations

**Entry Points:**
- `apps/web/app/layout.tsx`: root layout.
- `node apps/web/server.js`: standalone production server (generated by build).
- `apps/web/instrumentation.ts`: OTel registration.

**Configuration:**
- `apps/web/next.config.mjs`: standalone output toggle, transpilePackages, rewrites/redirects.
- `apps/web/Dockerfile`: multi-stage build → standalone runner.
- `docker-compose.coolify.yml` + `.env.example`: full self-host stack.
- `packages/env`: typed env.

**Core Logic:**
- `packages/web-backend/src/*`: Effect services.
- `packages/web-domain/src/*`: models/errors.

## Naming Conventions

**Files:** PascalCase for Effect services/domain (`Videos.ts`), kebab/route conventions for Next.js.
**Packages:** `@cap/<name>` workspace scope.

## Where to Add New Code

**New web feature/page:** `apps/web/app/<group>/...` + server action in `apps/web/actions/`.
**New API endpoint:** `apps/web/app/api/<name>/route.ts` (or extend Hono/Effect RPC router).
**New backend service:** `packages/web-backend/src/<Service>` + wire into `Rpcs.ts`/`index.ts`.
**DB change:** edit `packages/database/schema.ts`, generate migration (`pnpm db:generate`).

## Deploying the Web App (Coolify)

- **Build (Docker):** root `Dockerfile` at `apps/web/Dockerfile` builds the whole monorepo; sets `NEXT_PUBLIC_DOCKER_BUILD=true` → `pnpm run build:web` → Next.js `standalone`.
- **Output:** `apps/web/.next/standalone` (+ `.next/static`, `public`, `migrations`).
- **Server entry:** `CMD HOSTNAME="0.0.0.0" node apps/web/server.js`, `EXPOSE 3000`.
- **Prebuilt image:** `ghcr.io/capsoftware/cap-web:latest` (used by `docker-compose.coolify.yml`).
- **Required services:** MySQL 8, S3/MinIO, media-server (`ghcr.io/capsoftware/cap-media-server`).
- **Required env:** `DATABASE_URL` (mysql://), `WEB_URL`, `NEXTAUTH_URL`, `NEXTAUTH_SECRET`, `DATABASE_ENCRYPTION_KEY`, `CAP_AWS_*`/`S3_*`, `MEDIA_SERVER_WEBHOOK_SECRET`. Optional: `RESEND_*`, `DEEPGRAM_API_KEY`, `GROQ/OPENAI`, Google OAuth.
- **Note:** Cap is MySQL+Drizzle (not Postgres); deploy diverges from org Postgres default — keep MySQL for upstream parity.

## Special Directories

**`apps/web/.next/standalone`:** generated, not committed; produced by Docker build.
**`packages/database/migrations`:** committed SQL, copied into image, run via `migrate.ts`.
